A análise léxica é a primeira verificação feita sobre os programas escritos, basicamente separa o texto fonte em átomos / Tokens que podem ser utilizadaos nas etapas posteriores da compilação. Na análise léxica todo o conteúdo irrelevante para a compilação é descartado, como por exemplo: Espaços em branco, marcadores de nova linha, e comentários.
\subsection{Tokens}
A análise léxica do compilador se dá basicamente pelo reconhecimento dos Tokens definidos para a linguagem, são eles:
\begin{itemize}
	\item Identificador
	\item Número
	\item Palavra Reservada
	\item Caracter
	\item Comentário
	\item String
\end{itemize}

Para o reconhecimento dos Tokens(Átomos), lançou-se mão do uso de Autômatos finitos, que aceitam como entrada caracteres, e estando em um estado final, podem emitir um Token.

O token foi modelado com 3 campos: 

\begin{itemize}
	\item Valor
	\item Tipo
	\item Linha
\end{itemize}

\subsection{Automatos}
Os automatos para o reconhecimento dos Tokens foram modelados da seguinte forma, todos herdam da classe automata.java, {\it Código \ref{code:automata}}.
O campo State , representa o estado do Automato, os estados são membros de uma enumeração pública que tem, uma propriedade, se o estado é final ou não. {\it Código \ref{code:state}}

Cada um dos automatos implementa de forma diferente o método {\it processChar(char a)} que deve retornar um booleano ao final do método. O booleano a ser retornado deve ter valor verdadeiro caso o automato pode consumir o caracter fornecido, caso contrário falso será retornado e o analisador léxico deve verificar se o automato chegou a um estado final ou não , e pedir a emissão do Token caso o estado atual seja um estado final.

Dependendo do estado e de como ocorreu o processamento o automato pode ser solicitado a emitir um Token baseado no seu estado atual.
