A análise léxica é a primeira verificação feita sobre os programas escritos, basicamente separa o texto fonte em átomos / Tokens que podem ser utilizados nas etapas posteriores da compilação. Na análise léxica todo o conteúdo irrelevante para a compilação é descartado, como por exemplo: Espaços em branco, marcadores de nova linha, e comentários.
\subsection{Tokens}
A análise léxica do compilador se dá basicamente pelo reconhecimento dos Tokens definidos para a linguagem, são eles:
\begin{itemize}
	\item Identificador
	\item Número
	\item Palavra Reservada
	\item caracter
	\item Comentário
	\item String
\end{itemize}

Para o reconhecimento dos Tokens(Átomos), lançou-se mão do uso de automatos finitos, que aceitam como entrada caracteres, e estando em um estado final, podem emitir um Token.

O token foi modelado com 3 campos: 

\begin{itemize}
	\item Valor
	\item Tipo
	\item Linha
\end{itemize}

O campo State , representa o estado do Automato, os estados são membros de uma enumeração pública que tem, uma propriedade, se o estado é final ou não. {\it Código \ref{code:state}}

Cada um dos automatos implementa de forma diferente o método {\it processChar(char a)} que deve retornar um booleano ao final do método. O booleano a ser retornado deve ter valor verdadeiro caso o automato pode consumir o caracter fornecido, caso contrário falso será retornado e o analisador léxico deve verificar se o automato chegou a um estado final ou não , e pedir a emissão do Token caso o estado atual seja um estado final.

Dependendo do estado e de como ocorreu o processamento o automato pode ser solicitado a emitir um Token baseado no seu estado atual.

\subsection{Analisador Léxico}
O Analisador é a classe que tem o contato direto com o arquivo de texto fonte.
Sua função é obter os tokens do texto fonte a ser compilado, em sua estrutura possuí uma lista que contém todos os automatos, associado a cada automato, há um booleano, que indica se a aquele automato já retornou um booleano falso alguma vez, na análise do token atual.

O funcionamento do analisador léxico é descrito como : Ao ser iniciado, o analisador cria os meios para leitura de caracters um a um do arquivo fonte, e lê o primeiro caracter do arquivo.

Quando o método {\it getNextToken(table)} é chamado o analisador sintático executa os seguintes passos:

\begin{enumerate}
	\item Verifica se o último caracter é um caracter de espaço em branco (whitespace) , se for le o próximo até achar um que não seja.
	\item Verifica se o último caracter é o marcador de fim de arquivo, se sim, retorna o Token de EOF.
	\item Pede a todos os automatos para processarem o caracter, aqueles que retornarem falso são marcados como automatos desabilitados para o próximo caracter.
	\item Lê o próximo caracter
	\item Aos automatos restantes oferece o caracter para processamento.
	\item Verifica se ainda há automatos habilitados
	\item Se sim , volta a repetir desde o item 4.
	\item Se não há automatos habilitados, verifica o último automato a ser desativado
	\item Se esse automato está em estado final, recupera o token, reinicializa os automatos, e retorna o token
	\item Se o último automato não estiver em um estado final, isso caracteriza um estado de erro, o analisador sintático pára o processo, isso pode ocorrer por exemplo na presença de um caracter que não seja ascii no nome de um identificador, por exemplo.
	
\end{enumerate}

Se o token obtido no processo for um token de comentário, o analisador léxico descarta o token, e retorna o próximo token.

Se o token obtido for um identificador, o analisador léxico recupera o nome do identificador no automato de palavras e insere a palavra na tabela de símbolos passada, o índice retornado pela tabela de símbolo é o valor que o token vai assumir.
